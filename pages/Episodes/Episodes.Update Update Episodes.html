Cloud Radio

Update - Update Episode (User Token Required)

[PUT] /episodes/{id}

The id of the episode you want to update must be sent in the url as a query parameter
Example: If you wanna update the episode 1 you should use
http://www.cloudradio.net/episodes/1

Parameters

You must send a JSON Object containing the elements below

name: String, (optional)
description: String, (optional)
keywords: String, (optional)
duration: String, (optional)
soundwave: String, (optional)
number: String, (optional)
disqusThread: String, (optional)
fileSize: Integer, (optional)
markings: ObjectId, (optional)
categories: ObjectId, (optional)
image: ObjectId, (optional)
podcasters: ObjectId, (optional)
file: Object
path: String, (optional)
parts: Array[String], (optional)
bitrate: Number, (optional)
file64: Object,
path: String, (optional)
parts: Array[String], (optional)
file96: Object,
path: String, (optional)
parts: Array[String], (optional)
file128: Object,
path: String, (optional)
parts: Array[String], (optional)
publicationDate:

Understanding the fields:

name: The main name/title of the station
description: describe your station as best as you can
keywords: An array containing the keywords for the podcast
duration: The duration of the episode in seconds
soundwave: the soundwave of the episode, in hex
number: a unique identifier for yur episode as '1', '2'....'N'
disqusThread: Thread linked to your post in wordpress (used to sync disqus comments for each episode)
fileSize: sound file full size in Kbytes
markings: An array containing the ids of markings for this episode
categories: An array containing the ids of the categories
image: the id of the image previously updated
podcasters: An array containing the ids of all the users involved in the episode
file: an object containing the main filo info
path: the main file path
parts: an array containing the parts of the file, if it exist
bitrate: the bitrate of the audio file
file64: an object containing the main filo info
path: the main file path
parts: an array containing the parts of the file, if it exist
file96: an object containing the main filo info
path: the main file path
parts: an array containing the parts of the file, if it exist
file128: an object containing the main filo info
path: the main file path
parts: an array containing the parts of the file, if it exist
publicationDate: the date this episode must become public

Response

The response will contain two fields:

status: Containing 'success' or 'error
data: Containing the error message or the data created/updated/removed even the flag true
