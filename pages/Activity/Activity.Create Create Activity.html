What's considered an activity?

Almost everything is an activity (and watch out, our ist of supported activities grow larger as the system evolves). Below are listed the currently supported actions.

{action}

1 - like
2 - share
3 - dislike
4 - follow
5 - play

And the best of it all, you can use this actions for anything! Wanna share a user? Fell free. What about liking a station? There you go.

You must be thinking what kind of objects can i interact with? Well, here are the current options:

{objectType}

1 - user
2 - station
3 - podcast
4 - episode

And last but not least important, the id you wanna interact with:

{id}

Let's start using this activity's record then:

URL [POST] /activity/{action}/{objectType}/{id}

In order to record a new activity, you must send the parameters within the url, following the order above.

Example:

If you wanna like an episode: http://www.cloudradio.net.br/activity/like/episode/392817920

If you wanna follow a podcaster(or any other user):
http://www.cloudradio.net.br/activity/follow/user/8273218

Easy, right?

All we nedd you to send us also is the userId in a json in the request body:

userId: String, (required)

Warning

When creating an activity you'll add 1 to the total amount of it. For obvious purposes you can only play an episode.

Response

The response will contain two fields:

status: Containing 'success' or 'error
data: Containing the error message or the data created/updated/removed even the flag true
