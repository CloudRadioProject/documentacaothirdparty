Cloud Radio

Select- Get Trends (User Token Required)

What can i do with this route?

Using this route you can get the most popular objects being listened, shared and liked. By now you can only get the top trending using as filter object and action, but soon we will provide more options for you to acquire the perfect result.

So, let's start talking about the info we need to pass in order to get what we want. First, the type of action we wanna know about. There are 5 options of actions to filter:

{action}

like
follow
play
general

We also need to have object type we wanna get:

{objectType}

user
podcast
episode

Of course there will be a lot of results, that's why it is necessary for you to send in the query parameters (along with the url) the value for the limit and skip.

limit: ([skip] Integer, [amount] Integer), (required)

this parameter is composed of two fields separated by a "," :

[skip] : The amount of pages you wanna skip [amount] : the number of results you wanna get in this response So, if you send limit=30 you will get the first 30 results for your search If you send limit = 2,30 you will get the next 30 results (from 31 to 60) If you send limit = 3,30 you will get the next 30 results (from 61 to 90) and so on

Let's take a look on how this route will look like:

[GET] api.cloudradio.com.br:8080/trends/{action}/{objectType}

Example:

If you wanna get a list with the 10 most popular episodes in terms of listening you can use:
http://api.cloudradio.com.br:8080/trends/play/episode?limit=0,10

UPDATE

You can also specify now two query parameters: category and keyword.

Category: The category id you want to filter Keyword: An specific Keyword that the object shares (WARNING, that's the keywords, not title, not description)

Example:

http://api.cloudradio.com.br:8080/play/episode?limit=0,10&category=723jds78jbksd&keyword=esquilo

Response

The response will contain two fields:

status: Containing 'success' or 'error
data: Containing the error message or the data created/updated/removed even the flag true
