Cloud Radio

Select- Get Activity (User Token Required)

What can i do with this route?

Using this route you can acquire almost everything you need to know about the activities of an user, podcast, station and episode.

So, let's start talking about the info we need to pass in order to get what we want. First, the type of object we wanna know about:

{objectType}

1 - user
2 - station
3 - podcast
4 - episode
5 - history (sending this objectType you must send an userId in the field id, by doing this you will retrieve all data from that user, from the beggining of time, all of his "history" in CloudRadio)

We also need to have the specific id for the object:

{id}

And also, if you wanna know about a specific type of action, you may send a query parameter in the url as "action", there are 5 options of actions to filter:

{action}

1 - like
2 - share
3 - dislike
4 - follow
5 - play

Let's take a look on how this route will look like:

URL : [GET] /activity/{objectType}/{id}

Example:

If you wanna know everyone who liked your station:
http://www.cloudradio.net.br/activity/station/91821921?action=like

Wanna know everything your friend did in CloudRadio:
http://www.cloudradio.net.br/activity/history/2863283732

I think you got it.

Response

The response will contain two fields:

status: Containing 'success' or 'error
data: Containing the error message or the data created/updated/removed even the flag true
