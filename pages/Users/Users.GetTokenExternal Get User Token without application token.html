Auth - Get Token External

This route is used to get a logged user token without the need of an application token. It will mostly be used by our Wordpress Plugin. In order for a user to be able to login using this route it needs to enable it using this route

URL : [POST] /users/loginExternal

Parameters

You must send the parameters in the url.

Example: http://www.cloudradio.net/users/loginExternal?password=senhasegura&username=diego&userSecret=nevergonnagiveyouup

password: String, (required)
username: String, (required)
userSecret: String, (required)

Understanding the fields:

password: the user password
username: The user name informed at register
userSecret: A key generated when you enable external access in the user data

About Expiration

Each User token is valid for up to 60 minutes, after that you need to get a new token to continue using our API. The way to get a new token is making a call to this rout again, using the same parameters. Note that if your application token expires when you need to make a new call to get a user token, you can always get a fresh one.

Response

The response will contain two fields:

status: Containing 'success' or 'error
data: Containing the error message or the data created/updated/removed even the flag true
